Content

Skeleton

Before you should start writing the content of the Ebook, it is best to create a skeleton of a functional Ebook. This skeleton should contain a functional mimetype file that declares that the files in the folder constitute an Ebook. There should be a folder called META-INF that contains the container.xml file. Another requirement is the content.opf file in the OEBPS folder. This folder contains all the files that have content (e.g. xhtml files or pictures) or carry information about the style of the information (stylesheets). Every file that is in the OEBPS folder have to be listed in the manifest of the content.opf file. Further, all the content files have to be listed in the spine of the content.opf folder. They have to be listed in the right order. Then, it also helps to already prepare a file toc.xhtml that will contain the table of contents.

Adding Content

Every file that has content will have to be added to the manifest and the spine of the content.opf file. Further, the files will have to be added as a li element in the ol of toc.xhtml if they are supposed to show up in the table of contents.

Titlepage

The titlepage is made up of a picture and the title (and subtitle) in most Ebooks. The picture that has to be included will have to be listed in the manifest and spine of the content.opf file as well. It is important to center the images and put them in a container div. If pictures are not in container divs, they will not stretch according to the size of the window. The picture, the title and the subtitle all have to be part of the same div if you want to give them a common background color. In order to avoid page breaks in the title page, it is smart to set the attribute "page-break-inside" to "avoid" in the css file that describes the container div. You should also force a page break after the subtitle by setting the attribute "page-break-after" to "always" in the stylesheet.

Dedication

The dedication should be centered in the perfect middle of the page. For that, the text has to be centered horizontally by setting "text-align" to "center" in the stylesheet. For vertical centering, the "line-height" has to be set to 200px, and -after setting "margin" to "auto"- the "margin-top" has to be set to -100px (half ot the "line height"). Then, "top" and "width" have to be set to 50% and 100% respectively. The style of the text can be changed as pleased, in our case it was simply changed to italics.

Table of Contents and List of Illustrations

Simply create an ordered list in a nav element with the id "pub-toc" ("loi" for List of Illustrations), in which links to the content that is to be listed in the table of contents are added in the required order.

Acknowledgements, Preface, Chapters

Basically all function in the same way and share the same features. In our Ebook, we chose to have different headlines for chapters and the acknowledgments and preface, but that is up to the author of the Ebook and easily manipulated by CSS. The margins between the paragraphs were decreased for better readability. Quotes were introduced, by simply putting the text of the quotes into a div environment. The div has padding, so that the rounded borders that were drawn around the div do not interfere with the letters of the quote. By using span, very big quotation marks were put around the quotes. The font of the quotes was in italics. Images are, again, in a container div, so that they stretch with the window and centered. Captions are centered and in slightly smaller italic font.

Glossary

For better readability, we increased the margins between paragraphs in the glossary, because you usually do not want to read the whole page, only one word and its explanation are of interest. That is why they should be spatially separated from other words and their explanations in the glossary. Different text styles of the word and its explanation are created by using spans with different classes. The design of these classes can be manipulated as desired in CSS.

Colophon

In the colophon, the text should not fill out the whole page, but only a narrow block. In order to achieve this, the "width" of the paragraph simply had to be set to "12em" in CSS, and the margins had to be set to "auto". It is also important to specify that the text has to be aligned in the center. The image also has to be centered and, again, is placed inside a container div.

Page Breaks

For clearer page breaks, page breaks are forced or prevented in the following positions:

  • No page breaks inside container divs
  • Page break after the subtitle
  • Page Breaks before headings with the class "heading"
  • Page Breaks before images with the class "centeredimg"
  • No page breaks inside divs that contain quotes
  • Page breaks before chapter headings
  • Page breaks after captions of images
